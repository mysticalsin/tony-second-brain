# demo-brain — synthetic data so the dashboard is ALIVE on first open

Copy the two folders into your vault root right after the base install:

    cp -R starter/demo-brain/_brain_api starter/demo-brain/_agent_state <your-vault>/

Everything is fake and labeled [DEMO] (clients Globex/Initech, 8 toy agents).
The dashboard, pipeline tide, promises pulse, fleet patina, and spread tapes
all render with this seed — so you SEE the standard before wiring your own
data pipelines (specs/05). Wipe it any time:

    rm -rf <your-vault>/_brain_api && rm -rf <your-vault>/_agent_state

Your real `_brain_api` is regenerated by the pipeline tools; `_agent_state`
restarts clean. NO-THEATER NOTE: with neither demo nor real data, surfaces
render honest empty states — that is correct behavior, not breakage.
